# Spec: v2.16.0 — utag-factory (internal automation factory)

## Sources of truth

- `policies/autoscaling.yaml` — the operator's autoscaling template (imported from
  `~/Downloads/autoscaling.template.yaml`), promoted from draft to canonical by
  validating against the new `AutoscalingProfile` contract (the template's own
  stated requirement).
- Wolfram KB architecture notes: computation providers are hot-swappable
  adapters behind typed request/result envelopes; every expensive job carries a
  resource contract; the scheduler is telemetry-driven and instantiates a
  bounded number of workers regardless of how many logical agents exist.
- AgentForge / Clover / CoVe / Prompt Report: execution-grounded multi-role
  agents with mandatory sandboxing and dual vector retrieval — already
  implemented by `utag-veriforge`; the factory runs those agents as queue
  workers rather than reimplementing them.

## What v1 is

A daemonized, autoscaling job factory inside the utag monorepo
(`packages/python/utag-factory`), Python end-to-end:

- **Queue**: Redis Streams per the template — streams `utag:jobs|ingest|validate|
  evals|events|dlq`, consumer group `utag-workers`, at-least-once, pointers-only
  payloads, `XAUTOCLAIM` crash recovery.
- **State**: sqlite (WAL) job store + sqlite-vec vector store behind ports
  (`JobStorePort`/`VectorStorePort` names from the template).
- **Caches**: exact (Redis, 15m TTL) + semantic (sqlite-vec, 0.94 threshold,
  template key material). Embeddings behind an `EmbedderPort`; v1 default is a
  deterministic feature-hash embedder (offline, honest about being lexical),
  with the Flash `embed` tool as the real-model slot.
- **Retries/DLQ**: template taxonomy (retryable vs terminal), exponential-jitter
  backoff, max 3 attempts; DLQ entries carry the template's required fields;
  requeue only with validated overlay + passing regression eval.
- **Sandboxes**: bwrap default, plain-process fallback (capability-gated),
  extending veriforge's `Sandbox` protocol; remote Flash workers are the
  third isolation tier.
- **Workers/supervisor**: worker daemons claim jobs and execute typed job kinds
  (`run-python`, `generate-artifact`, `agent-task`); the supervisor implements
  the template's `desired_workers` formula, guardrails (memory reserve, drain
  before scale-down, failure-rate cap), and scale-to-zero. Daemonization via
  `systemd-run --user` transient units when available, plain subprocess
  otherwise. **Template amendment (decision of record): `execution.supervisor`
  is Python in v1; the Go supervisor remains a declared adapter target.**
- **Agents**: NL input → Instructor-structured job specs through the existing
  policy router (`utag_core.ai`); veriforge's five-role pipeline is the
  `agent-task` executor (experimental in v1).
- **Surfaces (operator's stack choice)**: **Typer** CLI (`utag-factory`),
  **FastAPI** (job submission/status + SSE event firehose from `utag:events`),
  **FastMCP** (factory operations as MCP tools for agents).
- **Flash `cloud_burst` adapter**: tool catalog (`run-python`, `embed`,
  `batch-generate`) as runpod-flash Endpoints with scale-to-zero autoscaling;
  used when the profile prefers remote over local (OOM risk) or a job demands
  it. Capability-gated on `RUNPOD_API_KEY`. **Weights volume + vLLM streaming
  endpoint: scaffolded, fast-follow — not in v1.**

## Phases

- **F0 contracts**: `AutoscalingProfile` + supporting schemas; import + validate
  the template; config loader with profile selection via `UTAG_AUTOSCALING_PROFILE`.
- **F1 substrate**: stores, queue, caches, retry/DLQ machinery (fakeredis-tested).
- **F2 execution**: sandboxes, worker loop, `run-python` + `generate-artifact`
  job kinds, run-evidence bridge into `utag:events`.
- **F3 scaling**: supervisor daemon, desired-workers formula, guardrails,
  systemd-run daemonization.
- **F4 agents**: NL → structured job (routed Instructor), `agent-task` kind.
- **F5 surfaces**: Typer CLI, FastAPI + SSE gateway, FastMCP server.
- **F6 burst**: Flash tool catalog + docs + release gates + PR.

## Non-negotiables (inherited)

Strict schemas with fixtures; no fake tests; capability-gated skips are visible
findings; run evidence for every job; no secrets in logs; deterministic where
promised; CHANGELOG per phase.
